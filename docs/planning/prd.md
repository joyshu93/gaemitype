# PRD Draft

## 문서 목적

개미타입 MVP의 제품 요구사항을 고정해 1인 개발자가 빠르게 구현과 배포까지 진행할 수 있도록 한다.

## 제품 요약

- 서비스명: 개미타입
- 한 줄 설명: 투자 습관과 판단 성향을 4축 16타입으로 가볍게 읽어주는 웹서비스
- 제품 성격: 재미와 자기이해 중심의 성향 테스트
- 비포함 성격: 투자 추천, 자산 배분 제안, 종목 선택 가이드, 금융 자문

## 문제 정의

개인투자자는 스스로의 투자 습관을 설명하기 어렵고, 자신의 행동 패턴을 가볍게 돌아볼 장치도 많지 않다. 개미타입은 이 틈을 메워 "나는 어떤 식으로 투자 판단을 하는 사람인가"를 짧고 재미있는 경험으로 보여준다.

## 목표

- 빠르게 완료 가능한 투자 성향 테스트를 제공한다.
- 사용자가 자기 유형을 이해하고 공유하고 싶게 만든다.
- 1인 개발자가 짧은 주기로 MVP를 완성하고 배포할 수 있는 구조를 유지한다.

## 비목표

- 실제 투자 의사결정 지원
- 종목/상품/포트폴리오 추천
- 사용자 계정 기반 히스토리 축적
- 커뮤니티 또는 소셜 네트워크 기능
- 실시간 시장 데이터 연동

## 대상 사용자

- 한국 주식 투자 경험이 있거나 관심이 있는 개인 사용자
- 투자 습관을 재미있게 돌아보고 싶은 사용자
- MBTI형 결과 공유 경험에 익숙한 사용자
- 너무 전문적인 금융 도구보다 가볍고 빠른 경험을 선호하는 사용자

## 핵심 가치 제안

- 쉬운 질문으로 투자 성향을 빠르게 읽을 수 있다.
- 결과가 직관적이고 기억하기 쉽다.
- 친구나 지인과 공유하기 좋다.
- 금융 자문이 아니라 부담 없는 자기이해 도구로 소비할 수 있다.

## 핵심 사용자 플로우

1. 사용자가 랜딩 페이지에 진입한다.
2. 서비스 목적과 비추천 원칙을 짧게 이해한다.
3. 테스트 시작 CTA를 누른다.
4. 20개의 질문에 답한다.
5. 점수 계산 후 4축 조합과 16타입 결과를 본다.
6. 결과 페이지에서 타입 설명, 강점, 주의점을 확인한다.
7. 결과를 링크 또는 이미지로 공유하거나 다시 테스트한다.

## 기능 요구사항

### 필수

- 랜딩 페이지
- 질문 플로우 페이지
- 진행률 표시
- 20문항 답변 수집
- 4축 점수 계산
- 16타입 결과 결정
- 결과 페이지
- 공유 영역
- 투자 조언이 아님을 명시하는 고지 문구

### 선택

- 결과 링크에 타입 코드 포함
- localStorage 기반 중간 진행 복원
- 공유용 이미지 카드

## 결과 페이지 요구사항

- 4글자 타입 코드
- 한글 별칭
- 핵심 한 줄 설명
- 4축 요약
- 강점 3개 내외
- 주의점 2개 내외
- 결과 해석 안내 문구
- 공유 CTA
- 다시 하기 CTA

## MVP 범위

- 로그인 없음
- DB 없음
- 정적 질문/결과 데이터 사용
- 정적 또는 프론트 중심 점수 계산
- 링크 기반 공유 우선
- 모바일/웹 반응형 대응

## 제외 범위

- 커뮤니티, 댓글, 팔로우
- 계좌 연동, 수익률 계산
- 실시간 차트, 시세, 뉴스 피드
- 개인 맞춤 투자 제안
- 결제, 유료 리포트
- 관리자 대시보드

## 성공 기준

- 랜딩 진입 사용자의 테스트 시작률이 충분히 높다.
- 시작 사용자 중 테스트 완료율이 높다.
- 결과 페이지에서 공유 또는 재도전 행동이 발생한다.
- 사용자 피드백에서 "내 얘기 같다", "공유하고 싶다" 반응이 나온다.
- 결과가 투자 추천처럼 보인다는 오해가 적다.

## 결정 사항과 이유

- 질문 수는 20개로 고정한다.
  - 이유: 축당 5문항으로 균형을 맞추기 쉽고, 체감상 아직 짧게 유지 가능하다.
- 결과는 4축 16타입 구조로 간다.
  - 이유: 공유성과 기억성, 구조적 확장성의 균형이 좋다.
- MVP는 프론트 중심 정적 구조를 채택한다.
  - 이유: 1인 개발자의 구현 및 배포 난이도를 낮춘다.
- 결과 표현은 성향 해석 중심으로 제한한다.
  - 이유: 투자 자문으로 오해될 가능성을 줄인다.

## 리스크

- 결과 카피가 추천처럼 읽힐 수 있다.
- 질문 문항이 비슷해 보이면 테스트 신뢰도가 떨어질 수 있다.
- 16타입 이름이 과하면 유치해 보일 수 있다.
- 공유 기능이 약하면 바이럴 동력이 떨어질 수 있다.

## 미확정 사항

- 결과 공유 이미지의 우선순위
- 타입별 상세 카피의 분량
- 결과 페이지에서 타입 비교 기능 포함 여부
- 배포 대상 도메인과 OG 이미지 정책

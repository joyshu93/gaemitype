# Type System Draft

## 목표

개미타입의 4축 16타입 구조를 일관되게 정의하고, 질문과 결과 카피가 같은 체계를 따르도록 만든다.

## 타입 시스템 원칙

- 각 축은 서로 다른 판단 차원을 다뤄야 한다.
- 좋은 타입과 나쁜 타입은 없다.
- 결과는 능력 평가가 아니라 행동 경향 설명이다.
- 타입명은 이해 가능성을 우선하고, 과도한 캐릭터화는 피한다.
- 결과 문구는 투자 조언처럼 읽히지 않아야 한다.

## 4축 정의

| 축 | 코드 | 해석 축명 | 설명 |
| --- | --- | --- | --- |
| 1 | A / P | 판단 준비 방식 | `A`는 기준과 시나리오를 먼저 정리하는 먼저 정리형, `P`는 흐름을 보며 대응을 조정하는 흐름 실행형 |
| 2 | L / T | 보는 시간 길이 | `L`은 긴 흐름과 보유 논리를 보는 긴 호흡형, `T`는 짧은 변동과 리듬에 빠르게 반응하는 빠른 반응형 |
| 3 | R / I | 판단을 움직이는 기준 | `R`은 자기 원칙과 체크 기준을 먼저 보는 원칙 체크형, `I`는 뉴스와 변화 신호를 먼저 보는 이슈 캐치형 |
| 4 | D / C | 확신을 얻는 방식 | `D`는 숫자와 자료를 확인해야 납득하는 데이터 확인형, `C`는 분위기와 흐름의 맥락을 읽으며 확신을 쌓는 맥락 감지형 |

## 코드 순서

타입 코드는 항상 아래 순서로 고정한다.

`준비 방식 -> 시간 감각 -> 판단 트리거 -> 근거 선호`

예시: `ALRD`

## 16타입 체계

| 코드 | 한글 별칭 | 핵심 설명 |
| --- | --- | --- |
| ALRD | 정교설계형 | 기준을 세우고 숫자로 확인한 뒤 길게 가져가는 타입 |
| ALRC | 원칙안정형 | 정한 룰을 바탕으로 차분하게 길게 보는 타입 |
| ALID | 정보분석형 | 새 정보와 숫자를 함께 검토하며 오래 보는 타입 |
| ALIC | 관찰축적형 | 흐름과 맥락을 천천히 모아 판단하는 타입 |
| ATRD | 빠른검증형 | 짧은 구간에서도 기준과 숫자를 빠르게 확인하는 타입 |
| ATRC | 유연점검형 | 기준은 지키되 상황에 맞게 가볍게 조정하는 타입 |
| ATID | 이슈추적형 | 이슈가 뜨면 자료를 확인하고 빠르게 반응하는 타입 |
| ATIC | 민감흐름형 | 시장 분위기와 변화 신호를 빠르게 읽는 타입 |
| PLRD | 차분실행형 | 흐름에 맞춰 움직이되 원칙과 숫자는 챙기는 타입 |
| PLRC | 안정조율형 | 큰 방향은 느긋하게 보면서 맥락에 맞춰 조율하는 타입 |
| PLID | 자료탐색형 | 유연하게 시작하지만 정보와 근거를 깊게 모으는 타입 |
| PLIC | 맥락관찰형 | 전체 흐름을 오래 보며 분위기 변화에 민감한 타입 |
| PTRD | 기민판단형 | 짧은 구간에서도 원칙과 숫자를 빠르게 맞추는 타입 |
| PTRC | 흐름조율형 | 상황을 보며 대응하되 기준은 놓치지 않는 타입 |
| PTID | 속도분석형 | 이슈가 생기면 빠르게 자료를 모아 판단하는 타입 |
| PTIC | 감각반응형 | 분위기와 변화에 민첩하게 반응하는 타입 |

## 타입 결과 공통 포맷

- 4글자 코드
- 한글 별칭
- 핵심 한 줄 설명
- 왜 이런 타입인지 설명
- 강점 3개
- 주의점 2개
- 성향 참고용 안내 문구

## 축별 강점 표현 가이드

- A: 준비가 빠르다, 기준을 세운다
- P: 변화에 유연하다, 상황 적응이 빠르다
- L: 긴 호흡을 유지한다, 흔들림에 둔감하다
- T: 반응 속도가 빠르다, 짧은 구간 변화에 민감하다
- R: 룰을 잘 지킨다, 자기 기준이 분명하다
- I: 새 흐름을 잘 포착한다, 변화 감지가 빠르다
- D: 숫자와 자료를 꼼꼼히 본다
- C: 분위기와 맥락을 넓게 읽는다

## 축별 주의점 표현 가이드

- A: 준비에 시간이 오래 걸릴 수 있다
- P: 기준이 자주 바뀔 수 있다
- L: 대응 속도가 늦어질 수 있다
- T: 잦은 판단 전환으로 피로가 생길 수 있다
- R: 새로운 흐름을 늦게 받아들일 수 있다
- I: 순간 이슈에 흔들릴 수 있다
- D: 과도한 확인으로 결정이 늦어질 수 있다
- C: 느낌이 근거보다 앞설 수 있다

## 점수 계산 원칙

- 축당 5문항을 배치한다.
- 각 문항은 한 축만 측정한다.
- 응답은 양극 선택으로 설계한다.
- 축별 다수값으로 최종 코드를 결정한다.
- 동점이 발생하면 더 강한 응답을 준 문항 수를 우선 사용한다.
- 동점 처리 규칙은 구현 문서에서 별도 명시한다.

## 결정 사항과 이유

- `D/C` 축은 분산/집중 대신 데이터/맥락으로 재정의한다.
  - 이유: 기존 `D/C = Diversified/Concentrated`는 실제 행동 추천처럼 읽힐 위험이 있고, 다른 축과 겹칠 여지가 있다.
- 타입명은 설명형을 우선 채택한다.
  - 이유: 첫 MVP에서는 공유보다 이해성과 안전성이 더 중요하다.
- 축 표기는 유지하되, 사용자에게 보이는 축 이름은 투자 행동 장면이 떠오르도록 보정한다.
  - 이유: 구조를 바꾸지 않고도 일반 성격검사처럼 보이는 인상을 줄일 수 있다.

## 미확정 사항

- 16타입 별칭을 더 감각적인 브랜드형으로 바꿀지 여부
- 동점 처리 기준의 세부 로직
- 축별 강약 표시를 결과 페이지에 노출할지 여부

# Question Spec Draft

## 목표

20개 질문을 4축에 균등 배치해 결과 판정의 일관성을 확보하고, 사용자 입장에서는 짧고 답하기 쉬운 질문 흐름으로 느껴지게 만든다.

## 질문 설계 원칙

- 총 20문항
- 축당 5문항
- 한 질문은 한 축만 측정
- 2지선다 또는 양극 선택형 문장 사용
- 금융 전문용어 최소화
- 정답처럼 보이는 표현 금지
- 사용자가 실제 행동을 떠올릴 수 있는 상황형 질문 우선

## 답변 형식 제안

- 기본형: 2지선다
- 대안형: 5점 스케일

MVP에서는 구현 단순성을 위해 2지선다를 우선 채택한다.

## 축별 질문 초안

### A / P 준비 방식

1. 새로운 정보를 보면 먼저 기준부터 세우는 편인가요, 일단 흐름부터 보는 편인가요?
2. 투자 전에 시나리오를 미리 정해두는 편인가요, 상황을 보며 바꾸는 편인가요?
3. 갑작스러운 변동이 와도 원래 계획을 우선 지키는 편인가요?
4. 시작 전에 비교표나 체크리스트를 만들어두는 편인가요?
5. 타이밍보다 준비 상태가 더 중요하다고 느끼나요?

### L / T 시간 감각

6. 한 번 잡은 관점은 오래 가져가는 편인가요, 짧은 구간에서 자주 조정하는 편인가요?
7. 조금 늦더라도 큰 그림이 맞으면 괜찮다고 보나요?
8. 오늘의 흔들림보다 앞으로의 방향이 더 중요하다고 느끼나요?
9. 손익 변화보다 보유 논리가 먼저인 편인가요?
10. 결과를 일주일보다 월 단위로 보는 편인가요?

### R / I 판단 트리거

11. 검증된 기준이 없으면 쉽게 움직이지 않는 편인가요, 이슈가 뜨면 먼저 확인하는 편인가요?
12. 남들이 많이 보는 정보보다 내 원칙이 우선인가요?
13. 변화가 생겨도 정한 룰을 지키려는 편인가요?
14. 익숙한 방식이 편한가요, 새 흐름이 보이면 판단을 바꾸는 편인가요?
15. 뉴스나 커뮤니티 반응을 보면 바로 확인하고 싶어지나요?

### D / C 근거 선호

16. 숫자와 자료를 먼저 보는 편인가요, 전체 분위기와 맥락을 먼저 보는 편인가요?
17. 말보다 자료가 있어야 납득되는 편인가요?
18. 복잡해도 쪼개서 확인해야 마음이 편한가요?
19. 비교 기준이 명확해야 판단이 쉬운가요?
20. 결과가 맞으면 과정은 어느 정도 유연해도 괜찮다고 느끼나요?

## 문항 작성 메모

- 3, 4, 5번 문항은 `A` 성향을 직접 측정하는 단문형이다.
- 7, 8, 9, 10번 문항은 `L` 성향이 안정적으로 드러나도록 설계한다.
- 11, 14, 15번은 `R/I` 경계가 가장 잘 드러나는 핵심 문항 후보다.
- 16번은 `D/C` 축의 대표 문항으로 결과 설명에도 재사용 가능하다.

## 점수 부여 방식 초안

- 각 질문은 두 선택지 중 하나를 해당 코드에 1점 부여한다.
- 축별 합산 후 높은 쪽 코드를 채택한다.
- 응답 데이터는 결과 계산 직후 폐기 가능하며, MVP에서는 서버 저장을 하지 않는다.

## 문항 품질 체크리스트

- 질문이 실제 행동 장면을 떠올리게 하는가?
- 두 선택지 중 어느 쪽도 더 좋아 보이지 않는가?
- 금융 자문이나 실력 평가처럼 읽히지 않는가?
- 문장 길이가 모바일 화면에서 과도하게 길지 않은가?
- 같은 축의 다른 문항과 중복 느낌이 과하지 않은가?

## 결정 사항과 이유

- MVP는 2지선다를 우선 채택한다.
  - 이유: 반응 속도와 구현 난이도 측면에서 가장 효율적이다.
- 문항은 축 설명보다 상황 묘사를 우선한다.
  - 이유: 사용자가 자연스럽게 답하도록 유도하기 쉽다.
- 문항 문구는 일반 성격 표현보다 실제 투자 장면이 떠오르도록 다듬는다.
  - 이유: 서비스 목적이 투자 습관 테스트임을 더 선명하게 보여주기 위해서다.

## 미확정 사항

- 2지선다 문구를 더 구어체로 다듬을지 여부
- 일부 문항에 예시 상황을 추가할지 여부
- 결과 정확도 향상을 위한 역문항 필요 여부

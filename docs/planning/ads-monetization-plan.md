# 개미타입 광고 수익화 기획안

## 문서 목적

이 문서는 개미타입의 1차 광고 수익화 방향을 정리한다.

광고의 목적은 `소규모 운영비를 보조하는 수준의 수익화`이며, 서비스의 핵심 경험인
`질문 -> 결과 -> 공유` 흐름을 해치지 않는 것이 최우선이다.

개미타입은 투자 추천 서비스가 아니므로, 광고 역시 결과 해석과 섞여
`맞춤 투자 제안`처럼 읽히지 않도록 강하게 분리해야 한다.

## 목표

- 결과 경험을 해치지 않는 선에서 광고 수익화 가능 상태를 만든다.
- 질문 완주율과 결과 공유율을 가능한 한 보존한다.
- 광고를 붙여도 서비스가 광고 사이트처럼 보이지 않게 유지한다.
- 운영자가 추후 광고 on/off와 위치를 제어할 수 있는 구조를 준비한다.

## 비목표

- 질문 흐름 안에서 광고 수익을 극대화하는 것
- 페이지당 여러 광고 슬롯을 운영하는 것
- 투자/금융 카테고리 광고를 적극적으로 노출하는 것
- 첫 라운드에서 광고 최적화 실험까지 모두 하는 것

## 1차 의사결정

### 최종 결정

- 1차 광고 도입 위치는 `결과 페이지 최하단 1개 슬롯`으로 제한한다.
- 홈 화면 광고는 `선택적 2차 실험`으로 남긴다.
- 질문 페이지와 계산 페이지에는 광고를 넣지 않는다.
- 광고 네트워크는 `Google AdSense`를 우선 검토한다.
- 자동 배치 전면 허용보다는 `제어 가능한 단일 슬롯`을 우선한다.

### 결정 이유

- 결과 페이지는 광고가 있어도 질문 완주율에 직접 영향을 주지 않는다.
- 결과 상단이나 중간 광고는 결과 해석 신뢰도를 해친다.
- 질문 중간 광고는 완주율을 떨어뜨릴 가능성이 높다.
- 개미타입은 결과를 공유하고 싶은 서비스이지, 광고 클릭을 유도하는 서비스가 아니다.

## 페이지별 광고 정책

### 홈 (`/`)

- 1차 범위에서는 광고 미적용
- 2차 실험 시에만 `하단 1개 슬롯` 검토
- 시작 CTA 위/옆/첫 화면 내 광고 금지

### 질문 (`/quiz`)

- 광고 금지
- 이유:
  - 질문 몰입 방해
  - 완주율 하락 가능성
  - 테스트 서비스가 설문 수익 페이지처럼 보일 위험

### 계산 (`/calculating`)

- 광고 금지
- 이유:
  - 짧은 전환 구간이며 광고 의미가 없음
  - 체감상 끊김만 늘릴 가능성이 큼

### 결과 (`/result`)

- 1차 광고 적용 대상
- 배치 위치:
  - 결과 카드
  - 4축 요약
  - 강점/주의점/습관
  - 안내 문구 및 공유 CTA
  - 위 요소가 모두 끝난 뒤 최하단

## 절대 금지 원칙

- 결과 카드 바로 아래 광고 배치 금지
- 4축 카드 사이 광고 배치 금지
- 공유 버튼과 광고를 시각적으로 섞는 디자인 금지
- 투자 추천, 종목 추천, 수익 보장, 급등, 자동매매 뉘앙스 광고와 결과 해석을 맞물리게 배치 금지
- 광고가 결과 일부처럼 보이게 하는 문구/장식 금지

## 광고 유형 가이드

### 피해야 할 광고

- 리딩방
- 코인 급등
- 고수익 보장
- 자동매매
- 레버리지/대출
- 투자 자문처럼 읽히는 금융 광고

### 상대적으로 안전한 방향

- 일반 웹서비스/생활 카테고리 광고
- 서비스 경험을 침범하지 않는 범용 광고
- 결과 해석과 직접 연결되지 않는 광고

## 1차 출시 범위

- 결과 페이지 최하단 광고 슬롯 1개
- 광고 on/off 환경변수
- 광고 라벨 명시
- 광고 차단기/미로드 상태 fallback
- 모바일 레이아웃 점검

## 2차 후보

- 홈 화면 하단 슬롯 1개 실험
- 광고 성능/이탈률 비교 후 유지 여부 판단
- 이후 필요 시 자동 광고 또는 추가 슬롯 재검토

## 성공 판단 기준

광고 도입 후 아래 지표를 비교한다.

- 테스트 시작률
- 질문 완주율
- 결과 페이지 도달률
- 결과 페이지 체류 시간
- 링크 복사율
- 공유 링크 재진입률
- 광고 도입 전후 모바일 이탈률

핵심 판단 기준은 `광고 수익`보다 `핵심 흐름 손상 여부`다.

## 단계별 롤아웃

### 0단계: 기준선 확보

- 광고 없는 현재 상태의 완주율/결과 도달률/공유율 기록

### 1단계: 결과 하단 단일 슬롯

- 결과 페이지 최하단에만 광고 노출
- 모바일 레이아웃 확인
- 공유 결과 모드에서도 결과 해석 이후에만 노출

### 2단계: 지표 검토

- 결과 페이지 체류 시간 변화
- 완주율 변화 여부
- 공유 액션 감소 여부

### 3단계: 홈 하단 확장 여부 판단

- 1단계 지표가 안정적일 때만 검토

## 구현 전 선행 조건

- 현재 결과 페이지 UX가 안정돼 있을 것
- 공유 링크와 직접 진입 UX가 정상일 것
- 모바일 결과 페이지 가독성이 확보돼 있을 것
- 결과가 투자 조언으로 오해되지 않는 문구 기준이 유지될 것

## 출시 전 수동 확인 포인트

- 광고가 결과 핵심 정보보다 먼저 보이지 않는가
- 모바일 첫 화면에서 광고가 CTA나 결과 해석을 밀어내지 않는가
- 광고 차단기 환경에서도 레이아웃이 깨지지 않는가
- 광고가 공유 버튼과 섞여 보이지 않는가
- 공유 결과 모드에서도 광고가 결과 일부처럼 보이지 않는가

## 보류한 대안

### 질문 중간 광고

- 보류 이유:
  - 질문 몰입 저해
  - 완주율 하락 위험
  - 브랜드 신뢰도 하락 가능성

### 결과 페이지 상단 광고

- 보류 이유:
  - 결과 해석보다 광고가 먼저 보이는 구조는 서비스 정체성과 충돌

### 전면 Auto ads

- 보류 이유:
  - 구현은 빠르지만 배치 제어력이 낮음
  - 1차 수익화보다 제품 신뢰가 더 중요함

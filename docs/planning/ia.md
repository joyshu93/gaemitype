# IA Draft

## 목표

개미타입의 핵심 흐름을 짧고 명확하게 유지하면서, 랜딩에서 결과 공유까지 끊기지 않는 사용자 경험을 만든다.

## 전체 구조

1. 랜딩 페이지
2. 질문 플로우 페이지
3. 계산/전환 상태
4. 결과 페이지
5. 공유 액션

## 페이지별 정보 구조

### 1. 랜딩 페이지

- 헤드라인
- 서브카피
- 테스트 시작 CTA
- 서비스 설명
- 결과 예시 미리보기
- 비추천 원칙 고지

### 2. 질문 플로우 페이지

- 상단 진행률
- 질문 카드
- 2지선다 또는 양극 선택 UI
- 이전 버튼
- 다음 자동 진행 또는 다음 버튼
- 중간 이탈 완화 문구

### 3. 계산/전환 상태

- 짧은 로딩 메시지
- 타입 계산 중 안내
- 결과 기대감 문구

### 4. 결과 페이지

- 타입 코드
- 한글 별칭
- 핵심 한 줄 설명
- 4축 결과 카드
- 왜 이런 타입인지 설명
- 강점
- 주의점
- 참고용 안내 문구
- 공유 버튼
- 다시 하기 버튼

### 5. 공유 액션

- 링크 복사
- 이미지 저장 또는 공유 카드
- 공유용 한 줄 카피

## 사용자 흐름 원칙

- 랜딩에서 "무엇을 하는 서비스인지" 5초 안에 이해 가능해야 한다.
- 질문은 한 화면 1문항으로 고정한다.
- 진행률은 항상 보이게 한다.
- 질문별 레이아웃은 통일한다.
- 결과 페이지는 점수 숫자보다 타입 정체성이 먼저 보이게 한다.
- 공유는 결과 확인 직후 가장 자연스럽게 노출한다.

## 질문 UX 원칙

- 질문 문장은 생활어 중심으로 짧게 쓴다.
- 금융 전문용어는 가능하면 피한다.
- 사용자가 정답을 찾으려 하지 않도록 가치판단 어휘를 줄인다.
- 초반 질문은 직관적이고 가벼운 문항으로 시작한다.
- 후반 질문은 미세한 차이를 가르는 문항을 배치한다.

## 결과 UX 원칙

- 상단 1스크린 안에 타입 정체성이 드러나야 한다.
- 4축은 막대그래프보다 카드형 요약을 우선 검토한다.
- 장점과 주의점은 행동 지시가 아니라 성향 해석 수준으로 제시한다.
- 고지 문구는 작게 숨기지 말고 자연스럽게 읽히는 위치에 둔다.

## 결과 페이지 구조 상세

### 상단

- `ALRD`
- `정교설계형`
- `기준을 먼저 세우고 오래 가져가는 편`

### 중단 1

- 4축 카드 4개
  - 준비 방식
  - 시간 감각
  - 판단 트리거
  - 근거 선호

### 중단 2

- 당신은 왜 이런 타입인가
- 강점 3개
- 주의점 2개

### 하단

- 이 결과는 투자 조언이 아닌 성향 참고용 안내
- 링크 복사
- 공유 이미지 저장
- 다시 테스트하기

## 모바일 우선 기준

- CTA는 하단 엄지 영역에서 쉽게 누를 수 있어야 한다.
- 선택지는 스크롤 없이 한 화면에 들어오는 높이를 우선한다.
- 텍스트는 길어도 3줄 안팎에서 끊기도록 작성한다.
- 결과 페이지 첫 화면에 타입명과 한 줄 설명이 모두 보여야 한다.

## 결정 사항과 이유

- 질문은 1문항 1화면으로 간다.
  - 이유: 집중도와 모바일 가독성이 가장 안정적이다.
- 결과 상단에는 코드보다 별칭과 한 줄 설명을 더 강조한다.
  - 이유: 공유성과 이해성을 동시에 높인다.
- 공유는 결과 직후 링크 복사 우선으로 제공한다.
  - 이유: MVP 구현 비용 대비 효율이 높다.

## 미확정 사항

- 질문 전환 애니메이션 강도
- 공유 이미지 자동 생성 여부
- 유형 비교 화면 추가 여부
